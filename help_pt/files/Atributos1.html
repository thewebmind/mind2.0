Atributos

Bem, você também pode usar este padrão para especificar os atributos e características dessas entidades.

Neste caso, as palavras chaves de ligação serão utilizados novamente, mas desta vez, a segunda entidade deve ser especificada como um atributo. Para isso, Web Mind precisa reconhecer o tipo de atributo. A Mind Natural Language compreende o tipo para este atributo após o charactere ":", estando antes dele o nome do atributo. Note que se você especificar o mesmo atributo duas vezes, somente a segunda será criada com o seu tipo e características.

Após estas palavras que representam o tipo do atributo, que são também multi-lingue,o Web Mind interpreta algumas características extras entre os caracteres "(" e ")".

A primeira característica é o tamanho, sendo apenas necessária para o alguns atributos, dependendo do SGBD.

Repare que, todo o sistema que você está construindo depende do seu SGBD, mesmo que a maioria deles tenham sintaxes muito parecidas.

O segundo parâmetro referencia o valor padrão deste atributo, que pode ser uma fórmula. Esse parâmetro não é obrigatório.

O terceiro parâmetro, ou o segundo, quando o segundo parâmetro não foi enviado, é para algumas características específicas de seu SGBD, como AUTO_INCREMENT ou não nulo.

No final de cada linha, você pode colocar algum comentário, definido por "//". Tudo após estes caracteres será considerado como comentário para esse campo.


Exemplo:

[conteúdo extra] < entidade > < palavra chave de ligação > < nome do atributo >: < tipo do atributo > (< tamanho >, "< valor padrão >", < características específicas >)
